# CRM Sales Analytics Dashboard Requirement Documentation Project
### Project Overview

This project demonstrates Business Analyst skills through a CRM Sales Analytics Dashboard requirement documentation project. The main focus is to understand a sales reporting problem and convert it into clear business requirements, functional requirements, KPIs, user stories, acceptance criteria, process flow, and UAT checklist.

The project uses a sample CRM sales dataset of 2,000 records to support dashboard planning, reporting logic, and validation.

### Business Problem

The sales team manages leads from multiple sources such as website, referral, social media, email campaigns, and cold calls. The current reporting process depends on manual Excel reports, which makes it difficult for managers to track lead conversion, revenue, overdue follow-ups, sales representative performance, and reasons for lost leads.

Because of this, sales managers do not get a clear view of pipeline health and may miss important follow-ups or performance issues.

### Project Objective

The objective of this project is to prepare complete documentation for a CRM Sales Analytics Dashboard that helps business users:
- Track total leads, converted leads, and lost leads
- Monitor sales revenue and open pipeline value
- Identify overdue and pending follow-ups
- Analyze sales representative performance
- Understand lead source effectiveness
- Review lost reasons and improve sales decisions
- Validate dashboard logic through UAT test cases

### Project Outcome

This project shows how we support an analytics dashboard project from requirement understanding to validation. It highlights skills in business analysis, requirement documentation, KPI definition, reporting logic, user stories, acceptance criteria, process mapping, and UAT validation.

The project also shows how raw CRM sales data can be converted into clear dashboard requirements that support better sales tracking and decision-making.
